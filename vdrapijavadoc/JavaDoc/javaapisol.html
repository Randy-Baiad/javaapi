Installing ViewDirect API for Java on Solaris

Installing the ViewDirect API for Java for the purposes of developing an API application consists of four steps: transferring the API files to the desired location on the application server, specifying the required environment variables, configuring document servers, and verifying the default configuration settings. These steps are described in detail below.

To deploy an API application, simply add your application code to the vdrapi.war file and distribute this file as needed.

The minimum requirements for running ViewDirect API for Java on Solaris are as follows:

  • JDK version 1.3 or later
  • Solaris operating system version 8 or greater, running in either 32- or 64-bit mode, with patch cluster dated Oct. 3, 2002 or later installed
  • Transarc DCE Secure Core Services (only required for ViewDirect for Networks installations)
  • TCP/IP connectivity to MVS (only required for ViewDirect for MVS installations)

Notes:

  • A J2EE-compatible application server is required for using DocumentDirect for the Internet Administrator to create a configuration profile for your ViewDirect API for Java-based application. However, you do not need to run the ViewDirect API for Java itself on a J2EE-compatible application server.
  • You cannot run two programs implementing the ViewDirect API for Java on the same application server. Additionally, you cannot run DocumentDirect for the Internet and a program implementing the ViewDirect API for Java on the same application server. You can overcome these restrictions by installing separate instances of the application server on the same machine, with each application server instance running with a single Mobius application instance on a separate port.
  • If you upgrade your version of the ViewDirect API for Java, you do not need to recompile your client application. It will continue to run as normal. However, if you do recompile your application, the compiler will indicate that the ViewDirect exception is not handled if an application object calls any of the following methods:

    • com.mobius.vdrapi.IDocumentServerFactor.totalNumberInList()
    • com.mobius.vdrapi.IHasDescription.getDescription()
    • com.mobius.vdrapi.IPageList.Add()
    • com.mobius.vdrapi.Isession.detach()

To resolve this issue, ensure that the application either catches or throws the ViewDirect exception prior to recompiling.

Transferring the API Files to the Desired Location

  1. Create a directory called ddrint.
  1. Copy vdrapi.war from the DocumentDirect for the Internet installation CD to the ddrint directory. You can find the vdrapi.war file in the /Transfer/ViewDirectJavaAPI directory.
  1. Use a file extraction utility, such as WinZip, to extract the contents of vdrapi.war. 

Note: If you plan to incorporate DocumentDirect for the Internet or FCP features into your application, use ddrint.ear or ddrint.war instead of the vdrapi.war file. The ddrint.ear and ddrint.war files are located in the /Transfer/Combined directory.

Specifying the Environment Variables

To specify the required environment variables, perform the following steps:

  1. Create and export the MOBIUS_REGPATH environment variable to specify the full path name of mobius.reg. Type a command based on the general format shown in the following example:

    export MOBIUS_REGPATH=.../ddrint/mobius.reg
  1. Create and export the NLSPATH environment variable to specify the full path name of the ddrint/WEB-INF/lib directory. Type a command based on the general format shown in the following example:

    export NLSPATH=.../ddrint/WEB-INF/lib/%N
  1. Create and export the LD_LIBRARY_PATH environment variable to specify the location of the ddrint/WEB-INF/lib directory, which is the same directory path as NLSPATH. For example, type the following command:

    export LD_LIBRARY_PATH=<full path of ddrint location>/WEB-INF/lib
  1. Add the value of NLSPATH to the PATH environment variable, and export it. For example, type the following command:

    export PATH=$PATH:<full path of ddrint location>/WEB-INF/lib/%N
  1. Add the full path of ddrint/WEB-INF/lib/vdrapi.jar to the CLASSPATH environment variable. For example, type the following command:

    export CLASSPATH=$CLASSPATH:<full path of ddrint location>/WEB-INF/lib/vdrapi.jar

Configuring Document Servers

To configure the document servers with which your application communicates, use the DocumentDirect for the Internet Administrator utility, which you must install and run on an application server. Refer to Appendix A in the DocumentDirect for the Internet 2.1 User’s Guide for more information about application server requirements.

To configure document servers, perform the following steps:

  1. Open the DocumentDirect for the Internet Administrator by entering the URL https://HOSTNAME:PORT/ddrint/servlet/config in your Web browser, where HOSTNAME refers to the location of your Application server on which DocumentDirect for the Internet was installed. Click the Document Servers tab. Refer to Chapter 5 in the User’s Guide for more information about configuring document servers.
  1. Close the DocumentDirect for the Internet Administrator. The document server details are saved in mobius.reg. If you ran the Administrator on a separate machine, copy the mobius.reg file to the machine where you are installing the ViewDirect API for Java.

Verifying the Configuration Settings

On the machine where you are installing the ViewDirect API for Java:

  1. Use a text editor to open mobius.reg
  2. Verify the following entries in the [HKEY_LOCAL_MACHINE\Software\mobius\INI\<DocumentDirect for the Internet application name>.ini\DDRINT_LOCATIONS] section to ensure the application server generates the correct directory paths:

    ExternalDataLocation=<full path of the DocumentDirect for the Internet work directory>
    ProgDirectory=<full path of the DocumentDirect for the Internet WEB-INF/lib directory>
    URLReferenceLocation=<full URL of the DocumentDirect for the Internet work directory>
  3. Make any necessary changes, then save and close the file.
